글 페이지 디자인

글 페이지는 react-markdown을 이용한 마크다운으로 이루어져 있다. custom components를 이용해 react-markdown에서 마크다운의 각 컴포넌트의 스타일을 변경할 수 있다 (관련 내용은 https://github.com/remarkjs/react-markdown?tab=readme-ov-file#appendix-b-components 에서 자세히 살펴볼 수 있다).

// markdown renderer custom components example
<Markdown
  components={% raw %}{{
    // Map `h1` (`# heading`) to use `h2`s.
    h1: 'h2',
    // Rewrite `em`s (`*like so*`) to `i` with a red foreground color.
    em(props) {
      const {node, ...rest} = props
      return <i style={{color: 'red'}} {...rest} />
    }
  }}{% endraw %}
/>

전체적인 디자인과 색상의 느낌은 https://mysetting.io/과 https://velog.io/의 다크 모드를 참고했다. 해당 사이트들의 느낌을 위해 추가한 컴포넌트는 크게 아래와 같다. 전체 style 코드는 위의 github commit에서 확인해보자.

  1. heading에는 넘버링별 크기 지정 및 아래 밑줄
  2. p(일반 폰트)는 어두운 흰색(#181818)
  3. strong(강조)에는 밝은 흰색(#000000)
  4. link에는 cadetblue(#5F9EA0)
  5. blockquote(인용문)에는 cadetblue left border 및 진회색 배경색상

또한 기존의 syntax highlighting 테마도 변경해 주었다. 현재 syntax highlighting 기능은 react-markdown에서 추천한 react-syntax-highlighter의 Prism을 사용하고 있다. Prism에서 기본으로 제공하는 테마의 목록은 https://github.com/react-syntax-highlighter/react-syntax-highlighter/blob/master/AVAILABLE_STYLES_PRISM.MD 에서 확인 할 수 있다. 현재 블로그는 darcula가 선택되어있다. 완성된 글 페이지는 아래와 같다.

## Lists

Unordered

+ Create a list by starting a line with `+`, `-`, or `*`
+ Sub-lists are made by indenting 2 spaces:
  - Marker character change forces new list start:
    * Ac tristique libero volutpat at
    + Facilisis in pretium nisl aliquet
    - Nulla volutpat aliquam velit
+ Very easy!
